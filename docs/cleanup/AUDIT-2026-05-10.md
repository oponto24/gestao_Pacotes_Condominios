## Observações para Fase 2 (@architect + @po)

### Itens de risco que precisam validação cuidadosa

1. **C4 / E1 (`.env.example` AIOX):** Pode ser regenerado pelo `aiox init`. Validar se removê-lo quebra setup de novos contribuidores antes de tocar.
2. **B2 / D2 (`storage/`):** Antes de remover, conferir `infra/docker/docker-compose.yml` — se tiver `volumes: ./storage:/app/storage`, o diretório precisa existir (manter com `.gitkeep`).
3. **A6 / A7 (auditorias handoff em `docs/stories/`):** Conferir com user se já foram aplicadas. Status "AGUARDANDO" pode estar desatualizado — várias PRs recentes (#77-#91) podem ter executado parte dos fixes.

### Candidatos óbvios (BAIXO risco, alta confiança)

1. **C1 (`jsqr`)** — confirmado zero imports em todo o repo (src + tests + scripts).
2. **C2 (`pino-http`)** — idem.
3. **A4 (`pacote_teste.jpeg`, `teste.jpeg` na raiz)** — apenas mtime/blob, sem referência ativa.
4. **E8 (`tsconfig.tsbuildinfo`)** — cache de build commitado por engano. Adicionar ao `.gitignore` e remover do tracking.
5. **C3 (`@types/papaparse` em `dependencies`)** — fix sem efeito runtime, apenas higiene.

### Sugestões de consolidação (3 → 1)

- **Doc histórico:** `ideia_de_projeto.md` + `estudo_etiquetas_e_bipe.md` + `modelos_de_etiquetas/*.jpeg` → `docs/legacy/discovery-2026-05/` (1 pasta, contexto preservado para arqueologia futura sem poluir raiz)
- **QA specs:** `docs/qa/e2e-specs/` + `docs/qa/test-designs/` → `docs/qa/specs/` (1 pasta)
- **Auditorias:** `docs/stories/auditoria-*-handoff.md` → `docs/audits/` ou `docs/cleanup/handoffs/` (separar stories de roadmap das auditorias one-shot)

### Não tocar (falsos positivos comuns)

- `MEMORY.md` em `.claude/projects/.../memory/` — contexto persistente IA, não é doc redundante
- `.aiox-core/` inteira — framework L1/L2, protegido por deny rules
- `html5-qrcode` — dynamic-imported em `BarcodeScannerInput.tsx`, parece órfão a `grep` ingênuo mas é usado
- `sharp`, `qrcode`, `svix` — todos importados em código de runtime
- Prisma migrations existentes — históricas, nunca remover
- Stories Done — manter como evidência de evolução; eventual arquivamento em `docs/stories/done/` é opcional, não urgente
