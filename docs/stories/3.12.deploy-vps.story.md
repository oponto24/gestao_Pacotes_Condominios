## User Story

**Como** product owner Ponto 24,
**eu quero** o MVP rodando em uma VPS com HTTPS e domínio próprio,
**para que** eu possa abrir o app `https://<domínio>` no celular real, dar pra clientes testarem, e validar o fluxo end-to-end (PWA com câmera, IA, matching, etc.) — algo que `localhost` no Mac não consegue mostrar.

---

## Contexto

User confirmou:
- ✅ **VPS** disponível
- ✅ **Domínio** disponível
- ⚠️ **Clerk** apenas em dev mode (precisa promover pra prod ou aceitar dev key em prod por enquanto)
- ⚠️ **Anthropic** chave pessoal temporária (ok pra MVP, troca depois)

PWA exige HTTPS pra `getUserMedia` (câmera) e `navigator.vibrate`. Nginx + Let's Encrypt (certbot) resolve.

**Storage:** local filesystem dentro do container (já é). Volume Docker persiste fotos. S3/R2 fica para Epic 8.

**Workflow desta story (interativo):**

1. **Coleta de info do user:**
   - IP da VPS, usuário SSH, método (chave/senha)
   - Domínio escolhido (ex: `app.ponto24.com.br`)
   - Provedor DNS (Cloudflare, Registro.br, etc) — onde apontar A record
   - Credenciais do Clerk dev (vai funcionar em prod, com aviso)
   - Anthropic API key prod (ou seguir com a temporária)

2. **Provisionamento:**
   - Subir Docker + docker-compose na VPS
   - Setar firewall (ufw): só 22, 80, 443
   - Criar usuário não-root pra deploy
   - Pull do repo

3. **Config produção:**
   - `infra/docker/docker-compose.prod.yml` (postgres + redis + app + worker + nginx + certbot)
   - `infra/nginx/nginx.conf` (reverse proxy app:3000 + worker health)
   - `.env.prod` template (NÃO commitado — gerado via script + secrets do user)
   - `Dockerfile.prod` se necessário (ou reusa multistage existente)

4. **DNS:**
   - User aponta domínio → IP da VPS (A record)
   - Aguarda propagação (alguns minutos)

5. **Deploy primeiro:**
   - `docker compose -f docker-compose.prod.yml up -d --build`
   - `prisma migrate deploy`
   - Seed (super-admin + WhatsApp number placeholder)

6. **Certificado SSL:**
   - `certbot --nginx -d <domínio>` (dentro do container ou no host)
   - Renovação automática via cron

7. **Smoke real no celular:**
   - User abre `https://<domínio>` no celular
   - Login com `oponto24@gmail.com` (Clerk dev)
   - Cria condomínio + setor + unidade + morador (full Epic 2)
   - Vai em /chegada, tira foto de etiqueta real → IA → matching → confirma → organiza → status `aguardando_retirada`
   - Verifica DB que pipeline funciona ponta-a-ponta

8. **CI/CD básico (deferred):**
   - Tech debt — push to main → deploy via GitHub Action (Epic 8.4)
   - MVP: deploy manual via SSH + script

## Dev Notes

### Pattern reusado
- `infra/docker/docker-compose.yml` (dev) é a base
- Worker entrypoint já existe (`workers/index.ts`)
- `prisma migrate deploy` é o caminho prod (já testado em dev)

### Nova superfície
- **Primeiro deploy real** do projeto
- **Pattern Docker Compose prod** que pode ser reusado por outros sistemas Ponto24
- **Nginx + certbot** pattern reusável

### Decisões importantes

1. **Storage local na VPS** — pra MVP. S3/R2 fica para Epic 8 quando tiver mais volume.
2. **Postgres + Redis self-hosted** — sem custo de Neon/Upstash. Backup manual via `pg_dump` cron (tech debt).
3. **Sem CI/CD automático** — MVP usa deploy manual via SSH. GitHub Action vem na Epic 8.4.
4. **Clerk dev key em prod** — funciona, com warning. User decide quando promover (cobra $25/mês a partir de N usuários).
5. **Sem domínio multi-tenant ainda** — `<domínio>` único pro condomínio piloto. Multi-tenant via subdomain fica para Epic 8.

### Riscos

1. **VPS cai durante demo** — sem alta disponibilidade, single point of failure. Mitigação: backup diário do postgres + restore documentado.
2. **Quota Anthropic temporária esgota** — user troca por chave prod assim que possível.
3. **Foto upload >5MB no 4G** — timeout. Já temos compressão na 3.2 (pra ~300KB). Tech debt: progress bar.
4. **PWA não instala** — checar `manifest.webmanifest` carregando + service worker (não temos SW ainda; é tech debt da 8.x). Sem SW, o app abre no browser mas não offline.

### Sugestões para Dev Notes

1. **Volume bind** pra fotos: `./storage/pacotes:/app/storage/pacotes` — facilita backup direto no host.
2. **Logging:** centralizar em `./logs/` no host com bind, ou usar `docker logs` com rotation.
3. **Backup postgres**: cron container que roda `pg_dump > backup.sql` diário, retém 7 dias.
4. **Health endpoint**: `/api/health/db` + `/api/health/redis` já existem (story 1.x). Nginx pode usar pra healthcheck.
