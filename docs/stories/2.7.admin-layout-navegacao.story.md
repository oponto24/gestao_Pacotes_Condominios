## User Story

**Como** admin/síndico do condomínio,
**eu quero** uma navegação lateral consistente em todas as telas administrativas
(`/admin/*`) com menu de cadastros, indicador do condomínio ativo e logout,
**para que** eu consiga navegar rapidamente entre Pacotes, Unidades, Moradores
e Setores sem perder contexto, e o sistema sinalize claramente em qual condomínio
estou operando.

---

## Contexto

UX_SPEC §3.4 define `AdminLayout` como **sidebar + header + main**. Hoje as
únicas telas existentes são `/` (home) e `/super-admin/condominios` (story 2.1).
A partir das stories 2.2-2.4 (CRUDs Setor/Unidade/Morador) e Epic 6 (painel
admin pacotes) o admin vai navegar entre 5+ telas — precisamos de chassi
visual antes de criar mais páginas soltas.

**Esta story:**
1. Cria `src/app/(admin)/layout.tsx` (route group Next 15 — não afeta URL)
2. Cria componentes `AdminLayout`, `AdminSidebar`, `AdminHeader`, `UserMenu`
3. Aplica em `/admin/*` (criando rota stub `/admin` como dashboard placeholder)
4. **Mobile responsivo:** sidebar vira drawer (Sheet existente da 2.1)
5. Guard de role: apenas `admin` (super_admin redireciona para `/super-admin`,
   porteiro para `/portaria` quando existir, ou `/`)
6. Header mostra: nome do condomínio + nome do user + menu (perfil, logout)
7. Sidebar com items: **Pacotes** (placeholder hoje), **Cadastros** (Setores,
   Unidades, Moradores — hoje placeholders), **Sair**

**Decisões pré-aprovadas:**
- **Route group `(admin)`** em vez de `/admin/_layout.tsx` — convenção Next 15 mais limpa
- **Não criar telas reais de cadastro** nesta story — apenas placeholders com `EmptyState` ("Em breve — story 2.X"). Mantém PR enxuto e foca no layout
- **Logout via Clerk** — usa `<SignOutButton>` do `@clerk/nextjs` (já instalado)
- **Sem tema claro/escuro** nesta story — story dedicada se necessário
- **Sem breadcrumbs** — sidebar + título da página são suficientes pra MVP

**Out of scope:**
- Telas reais de Setores/Unidades/Moradores (stories 2.2-2.4)
- Tela `/admin/pacotes` (story 6.1)
- SuperAdminLayout (story 8.1)
- PortariaLayout (story 3.1)
- Notifications/toasts globais (story dedicada se necessário)
- Tema dark/light
- Atalhos de teclado
- Breadcrumbs
- Sidebar collapse persistido em localStorage
