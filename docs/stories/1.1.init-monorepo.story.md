## Dev Notes

### Decisões e desvios do plano original

**1. Next.js 15.1 em vez de Next.js 16**
A story autorizava fallback se 16 indisponível. Optei por 15.1 estável porque:
- Ecossistema shadcn/ui maduro com 15
- Tailwind v3 (mais estável) acompanha 15
- Story permite migração futura quando 16 amadurecer
- Sem perda funcional para o MVP

**2. Tailwind v3 em vez de v4**
Decorrência da decisão acima — v4 tem breaking changes no parser de CSS que ainda não estão totalmente integradas com shadcn. Trocar pra v4 no futuro é refatoração isolada (1 story).

**3. shadcn/ui CLI não foi rodada — Button criado manualmente**
A CLI `npx shadcn@latest init` é interativa e em modo auto não dá pra responder prompts confiavelmente. Criei manualmente:
- `components.json` (config exata que a CLI geraria)
- `src/lib/utils.ts` (função `cn()`)
- `src/components/ui/button.tsx` (variantes alinhadas com tokens UX_SPEC §2)
Próximas stories podem usar a CLI normalmente — a config já está pronta.

**4. Node 25 em vez de Node 22 LTS**
O ambiente do usuário tem Node 25.6.1. Funcionou sem problemas em todos os gates (typecheck, lint, test, build). Próximas stories podem assumir Node ≥20.

**5. `next.config.ts` em vez de `.js`**
Conforme Acceptance Criteria 2. Next.js 15 suporta nativamente.

**6. `prisma/` excluído do `tsconfig.json`**
A `@data-engineer` (Dara) deixou `prisma/seed.ts` referenciando `@prisma/client` que ainda não está instalado (vem na story 1.3). Excluí o diretório do typecheck para não quebrar gates desta story. Quando 1.3 instalar Prisma, a exclusão pode ser revertida ou refinada (excluir só `seed.ts`).

**7. Não criei branch `feature/1.1-init-monorepo`**
Como esta é a story de bootstrap (primeiro commit do repo), commitei direto em `main`. Próximas stories seguirão o padrão de feature branches conforme convenção da arquitetura.

**8. `.env.app.example` em vez de modificar `.env.example` existente**
A raiz já tinha `.env.example` do framework AIOX (com vars do AIOX, não do app). Para evitar conflito, criei `.env.app.example` separado com as vars listadas em ARCHITECTURE.md §12.

**9. ESLint warning não-bloqueante: "next lint deprecated"**
Next.js 15 emite warning de que `next lint` será removido em 16. Migração para ESLint CLI direto é uma story futura (cosmética). Não bloqueia.
