# Story 13.1 — Remover tempPassword de respostas JSON

**Epic:** E13 — Hardening UI/UX + Segurança Frontend (P0)
**Status:** Done
**Owner:** @dev (Dex)
**Tipo:** Security fix
**Severidade:** CRÍTICA
**Estimativa:** 0.5 dia

---

## User Story

**Como** super-admin,
**eu quero** que senhas temporárias nunca apareçam em respostas HTTP,
**para que** não haja risco de interceptação ou log de credenciais em plaintext.

---

## Contexto

POST `/api/admin/users` retorna `{ tempPassword: "..." }` no JSON de resposta. Isso viola OWASP A02:2021 (Cryptographic Failures). A senha temporária deve ser comunicada apenas via canal seguro (email).

---

## Acceptance Criteria

### AC1 — API não retorna tempPassword
- [x] POST `/api/admin/users` NÃO retorna `tempPassword` no body
- [x] Response retorna `{ ok: true, userId, passwordSentVia: 'email' }`

### AC2 — Envio por email
- [x] Se email do user existe, envia email com senha temporária
- [x] Se email não existe, retorna instrução para admin configurar manualmente via Clerk

### AC3 — UI atualizada
- [x] Componente que cria user mostra "Senha enviada por email" em vez de exibir a senha
- [x] Fallback: se envio falhou, mostra mensagem "Não foi possível enviar. Configure via painel Clerk."

### AC4 — Testes
- [x] Test: POST /api/admin/users response não contém campo `tempPassword`
- [x] Test: POST /api/admin/users response contém `passwordSentVia`
