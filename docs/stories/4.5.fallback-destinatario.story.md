## User Story

**Como** sistema,
**eu quero** escolher o destinatário da notificação seguindo prioridade: morador da unidade com nome que casa com a etiqueta → adicionais → condômino principal,
**para que** o pacote chegue para a pessoa certa, mesmo quando a etiqueta tem nome diferente do principal (filhos, dependentes, cônjuge).

---

## Contexto

**FR-031 (PRD):** "Notificação deve ser enviada prioritariamente para o nome do destinatário que aparece na etiqueta, se ele estiver cadastrado naquela unidade."
**FR-032:** "Se o destinatário não estiver cadastrado, notificação vai para o condômino principal da unidade (fallback)."
**FR-005:** "Cada unidade pode ter destinatários adicionais (nome, telefone opcional)."

**Já existe (story 3.7):** matching de etiqueta → unidade. O nome extraído está em `pacote.ia_extracao_raw.nome_destinatario`.

**O que falta:** dado pacote organizado, escolher **qual morador** (entre principal + adicionais da unidade) deve receber.

## Acceptance Criteria

### AC1 — Helper `chooseRecipient` em `src/lib/whatsapp/recipient.ts`
- [x] Assinatura: `chooseRecipient(pacoteId: string): Promise<Recipient | null>`
- [x] `Recipient = { morador_id: string; nome: string; telefone: string; matched_by: 'nome_etiqueta' | 'principal' | 'fallback_adicional' }`

### AC2 — Algoritmo
1. Carrega pacote → unidade → todos os moradores da unidade (principal + adicionais com `ativo = true`)
2. Lê `pacote.ia_extracao_raw.nome_destinatario` (string ou null)
3. Se nome existe:
   - Normaliza (lowercase, sem acento, trim)
   - Para cada morador, normaliza nome
   - Match strategy:
     - **Exato (após normalização)** → retorna com `matched_by: 'nome_etiqueta'`
     - **Fuzzy** (Levenshtein normalizado <0.3 OU primeiro+último nome batem) → retorna com `matched_by: 'nome_etiqueta'`
4. Se nenhum match no nome:
   - Se principal tem telefone → retorna principal com `matched_by: 'principal'`
   - Senão se algum adicional tem telefone → retorna primeiro adicional com `matched_by: 'fallback_adicional'`
5. Se ninguém tem telefone → retorna `null`

### AC3 — Função pura testável
- [x] Lógica de matching de nome fica em função separada `matchByName(extractedName, moradores) → Morador | null`
- [x] Sem I/O — só normalização e comparação
- [x] Testes unitários cobrem casos:
  - "Maria Silva" vs "Maria da Silva" (acerto fuzzy)
  - "JOÃO" vs "João Silva" (acerto fuzzy primeiro nome)
  - "Maria" vs "Pedro" (não acerto)
  - "Maria Silva" vs "Maria Souza" (não acerto — sobrenomes diferentes)
  - acentuação: "JOSÉ" vs "Jose" (acerto)
  - Capitalização: "PEDRO COSTA" vs "pedro costa" (acerto)
