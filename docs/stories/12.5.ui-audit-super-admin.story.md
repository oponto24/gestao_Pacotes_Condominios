## User Story

**Como** super-admin,
**eu quero** visualizar o historico completo de auditoria com filtros interativos e detalhes de diff,
**para que** eu possa investigar qualquer alteracao feita no sistema, por quem e quando.

## Acceptance Criteria

- [x] AC1: Filtros interativos na page `/super-admin/audit`
  - Select de acao (todos, impersonate_start/stop, *_created, *_updated, *_deleted)
  - Select de condominio (todos + lista de condominios)
  - Select de entidade_tipo (todos, condominio, user, morador, unidade, setor, bloco)
  - Date range (de/ate) com inputs nativos date
  - Busca por user (nome ou email)
  - Filtros via URL search params (pattern existente)
- [x] AC2: Paginacao funcional
  - Botoes "Anterior" / "Proxima" com estado disabled correto
  - Indicador "Pagina X de Y" ou "Mostrando 1-20 de 150"
  - Default 20 por pagina
- [x] AC3: Tabela/lista expandida com novas acoes do 12.4
  - Labels para todas as acoes: bloco_created/updated/deleted, morador_created/updated/deleted, setor_*, unidade_*, user_created/updated/deleted
  - Icones diferenciados por tipo de acao (create=verde, update=azul, delete=vermelho)
- [x] AC4: Viewer de diff para metadata before/after
  - Quando metadata contem `before` e `after`, renderizar diff visual (campo: antes → depois)
  - Campos alterados (`changed`) destacados
  - Quando metadata contem apenas `after` (create), mostrar snapshot
  - Quando metadata contem apenas `before` (delete), mostrar snapshot com indicador "deletado"
- [x] AC5: Filtro de user_id funcional na API
  - Adicionar filtro `user_id` ou `user_q` (busca por nome) no helper `listAuditLogs`
- [x] AC6: Testes unitarios + quality gates
  - Lint, typecheck, build OK

## Decisoes tecnicas

1. **Expandir componente existente:** `AuditLogList` ja existe, expandir com filtros e diff viewer
2. **Client component para filtros:** Filtros precisam de interacao, usar `'use client'` em componente de filtros
3. **Server component para dados:** Page continua server component, filtros via search params
4. **Diff viewer:** Componente `AuditDiffViewer` que renderiza `changed` fields com before/after side-by-side
5. **ACAO_LABEL map:** Expandir o map existente com todas as novas acoes do 12.4
