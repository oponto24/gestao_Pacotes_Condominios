# Story 11.5 — Filtros padronizados em listas admin

**Epic:** E11 — UX Admin Refinado
**Status:** InProgress
**Owner:** @dev (Dex)
**Estimativa:** 0.25 dia
**Branch:** `main`
**Depende de:** 11.2

---

## User Story

**Como** admin do condominio,
**eu quero** filtrar as listas de moradores, unidades e pacotes por bloco,
**para que** eu consiga localizar informacoes rapidamente em condominios com muitas unidades.

---

## Acceptance Criteria

### AC1 — Filtro de bloco em `/admin/moradores`
- [ ] Dropdown "Bloco" com lista de blocos do condominio
- [ ] Filtra moradores cuja unidade pertence ao bloco selecionado
- [ ] Combina com busca textual existente

### AC2 — Filtro de bloco em `/admin/unidades`
- [ ] Dropdown "Bloco" filtra unidades por bloco_id
- [ ] Busca textual por identificador (se nao existir)

### AC3 — Filtro de bloco em `/admin/pacotes`
- [ ] Dropdown "Bloco" filtra pacotes cuja unidade pertence ao bloco
- [ ] Combina com filtros existentes (status, periodo)

### AC4 — Padrao de implementacao
- [ ] Filtros via URL search params (pattern existente)
- [ ] Componente Select do shadcn/ui
- [ ] Opcao "Todos" como default (sem filtro)

### AC5 — Quality gates
- [ ] Lint, typecheck, test, build OK
