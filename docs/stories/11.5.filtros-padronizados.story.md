# Story 11.5 — Filtros padronizados em listas admin

**Epic:** E11 — UX Admin Refinado
**Status:** Done
**Owner:** @dev (Dex)
**Estimativa:** 0.25 dia
**Branch:** `main`

## User Story

**Como** admin do condominio,
**eu quero** filtrar as listas de moradores, unidades e pacotes por bloco,
**para que** eu consiga localizar informacoes rapidamente em condominios com muitas unidades.

---

## Acceptance Criteria

### AC1 — Filtro de bloco em `/admin/moradores`
- [x] Dropdown "Bloco" com lista de blocos do condominio
- [x] Filtra moradores cuja unidade pertence ao bloco selecionado
- [x] Combina com busca textual existente

### AC2 — Filtro de bloco em `/admin/unidades`
- [x] Dropdown "Bloco" filtra unidades por bloco_id
- [x] Busca textual por identificador (se nao existir)

### AC3 — Filtro de bloco em `/admin/pacotes`
- [x] Dropdown "Bloco" filtra pacotes cuja unidade pertence ao bloco
- [x] Combina com filtros existentes (status, periodo)

### AC4 — Padrao de implementacao
- [x] Filtros via URL search params (pattern existente)
- [x] Componente Select do shadcn/ui
- [x] Opcao "Todos" como default (sem filtro)

### AC5 — Quality gates
- [x] Lint, typecheck, test, build OK
