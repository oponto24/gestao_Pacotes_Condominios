**Status:** Done (retroativa)

**Tipo:** Hotfix (produção)

---

## User Story

**Como** sistema,
**eu quero** que o pipeline WhatsApp funcione end-to-end em produção,
**para que** moradores recebam notificações com QR code quando um pacote chega.

---

## Contexto

Após deploy, o pipeline WhatsApp tinha 5 falhas em cascata impedindo o envio de mensagens. Cada fix revelava o próximo problema. Correções feitas em hotfix direto na main (produção parada).

---

## Acceptance Criteria

### AC1 — BullMQ jobId sem caracteres inválidos
- [x] `enqueue-send-whatsapp.ts`: jobId de `sendWhatsApp:${id}` → `sendWhatsApp-${id}`
- [x] `process-whatsapp-webhook.ts`: jobId de `palavraChave:${id}` → `palavraChave-${id}`
- [x] BullMQ não rejeita mais jobs com erro "Custom Id cannot contain :"

### AC2 — Storage retorna URLs absolutas
- [x] `storage/local.ts` `publicUrl()` usa `APP_URL` env var para gerar URL absoluta
- [x] Meta Cloud API consegue fazer download da imagem QR

### AC3 — Rota pública de storage
- [x] `src/app/api/storage/local/[...path]/route.ts` criada
- [x] Serve apenas tipos de imagem conhecidos (png, jpg, webp) — security
- [x] Headers: Content-Type correto, Cache-Control public 24h
- [x] Buffer convertido com `new Uint8Array()` para compatibilidade BodyInit

### AC4 — QR code formato quadrado
- [x] `qr/generator.ts` mudou de 1200×628 (panorâmico) para 800×800 (quadrado)
- [x] QR centralizado, texto abaixo

### AC5 — Dockerfile storage dirs
- [x] Worker stage pré-cria `storage/qr` e `storage/pacotes` com `chown`
- [x] Sem mais EACCES em produção

### AC6 — Template palavra-chave fix params
- [x] `process-palavra-chave.ts` envia nome do morador (não código) como {{1}}
- [x] Busca morador via `withSuperAdmin` antes de enviar template
