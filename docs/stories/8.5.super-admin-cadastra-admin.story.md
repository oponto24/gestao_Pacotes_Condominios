## User Story

**Como** super-admin Ponto24,
**eu quero** cadastrar diretamente o primeiro admin de cada condomínio (sem fluxo de convite por email),
**para que** quando eu fechar com um cliente, possa deixar tudo pronto pra o admin entrar via Clerk e operar imediatamente (decisão produto 2026-05-07: cadastro direto).

---

## Contexto

**Decisão de produto:** cadastro direto (não convite por email com link).
- Super-admin digita email + nome + escolhe condomínio → sistema cria registro `User` pendente
- Quando a pessoa cria conta no Clerk com aquele mesmo email, webhook `user.created` reconcilia (já implementado na 1.5/1.10): atualiza `clerk_id` placeholder pelo real, mantém `role` e `condominio_id`

Reuso do pattern do super-admin no seed (`prisma/seed.ts`): `clerk_id='pending_clerk_link_<hash>'` enquanto não houver login.

**Esta story entrega:**
1. Tela `/super-admin/users` listando todos os admins por condomínio
2. Form "Adicionar admin" — escolhe cond + email + nome
3. API `POST /api/super-admin/users` cria User pendente
4. Audit log automático

**Fora do escopo:** porteiros (8.7), admin master vs comum (8.8 backlog), edição/desativação de user (tech debt).
