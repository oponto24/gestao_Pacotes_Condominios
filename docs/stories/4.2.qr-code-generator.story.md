## User Story

**Como** sistema,
**eu quero** gerar uma imagem PNG 1200×628 contendo o QR Code do pacote centralizado + nome do condomínio embaixo,
**para que** o WhatsApp não corte o QR no preview do header de template (aspect 1.91:1) e o morador consiga escanear direto da prévia.

---

## Contexto

**FR-033 (PRD):** "Mensagem deve conter: nome do condomínio em destaque, nome do morador, aviso de chegada, setor de retirada, **QR Code único** (imagem)."
**FR-034:** "QR Code não expira — vale até ser usado."
**FR-047:** "Sistema deve invalidar o QR Code após uso."

**Decisão técnica anotada (2026-05-08):**
WhatsApp força aspect 1.91:1 no preview do header. QR quadrado 512×512 fica esticado/cortado. Solução: canvas 1200×628 com QR centralizado + nome do condomínio em texto abaixo.

**Esta story entrega APENAS a função de geração + persistência.** O envio em si fica em 4.3.

**Conteúdo do QR:** URL absoluta `${APP_URL}/retirar/${pacote_id}/${qr_token}` onde `qr_token` é hash criptograficamente seguro armazenado no banco.

## Acceptance Criteria

### AC1 — Schema Prisma — campo `qr_token` em `Pacote`
- [ ] Adicionar `qr_token String @unique @db.VarChar(64)` em `Pacote`
- [ ] Adicionar `qr_image_path String? @db.VarChar(500)` (path no storage, nullable até primeira geração)
- [ ] Adicionar `qr_consumido_em DateTime?` (FR-047 — invalidação após uso)
- [ ] Migration `prisma/migrations/{timestamp}_add_qr_pacote/`
- [ ] Backfill: stories anteriores deixaram pacotes sem token; gerar UUID v4 base64url para registros existentes na migration

### AC2 — Helper `generateQrToken()`
Em `src/lib/qr/token.ts`:
- [ ] Usa `crypto.randomBytes(32).toString('base64url')` (URL-safe, 43 chars)
- [ ] Garante unicidade via retry (max 3) — colisão é praticamente impossível mas segurança em profundidade

### AC3 — Helper `generateQrImage`
Em `src/lib/qr/generator.ts`:
- [ ] Assinatura: `generateQrImage(input: { qrToken: string; condominioNome: string; pacoteId: string }): Promise<{ buffer: Buffer; mimeType: 'image/png' }>`
- [ ] Lib: `qrcode` (já no package.json se existir; senão adicionar) + `sharp` (composição)
- [ ] Pipeline:
  1. Gera QR PNG 600×600 com margem 4 módulos, error correction `H` (alta — sobrevive a artefatos de compressão WhatsApp)
  2. Cria canvas 1200×628 fundo branco via `sharp`
  3. Composita QR centralizado verticalmente, deslocado pra esquerda (left=80, top=14)
  4. Renderiza texto "{condominioNome}" + label "Apresente para retirar" via SVG embutido (font system, peso 600, tamanho 38 pra nome / 22 pra label) — composita à direita
  5. Output PNG comprimido `compressionLevel: 9`
- [ ] Tamanho final esperado: <100KB

### AC4 — Storage upload em `src/lib/qr/store.ts`
- [ ] `saveQrImage(condominioId: string, pacoteId: string, buffer: Buffer): Promise<{ storagePath: string; publicUrl: string }>`
- [ ] Path: `qr/{condominio_id}/{pacote_id}.png`
- [ ] Usa `storage.put()` (story 1.9)
- [ ] Retorna URL pública (em dev: `/storage/qr/...`; em prod: URL assinada com TTL 7 dias)
- [ ] **Importante:** URL precisa ser acessível pela Meta (download direto) — em dev local Meta NÃO consegue. Documentar que smoke real só roda com APP_URL público (ngrok/cloudflared/staging).

### AC5 — Helper top-level `ensureQrForPacote`
Em `src/lib/qr/index.ts`:
- [ ] Assinatura: `ensureQrForPacote(pacoteId: string): Promise<{ qrToken: string; publicUrl: string }>`
- [ ] Idempotente: se pacote já tem `qr_image_path`, retorna existente
- [ ] Senão: gera token (se ausente) → gera imagem → salva storage → atualiza pacote → retorna URL
- [ ] Tudo em transação Prisma

### AC6 — Endpoint público `/api/qr/[token]/redirect`
- [ ] Route handler em `src/app/api/qr/[token]/redirect/route.ts`
- [ ] GET recebe `token`, busca pacote por `qr_token`, redireciona 302 pra `/retirar/{pacote_id}` (story Epic 5)
- [ ] Se token não existe → 404
- [ ] Se token consumido → 410 Gone com mensagem amigável
- [ ] Não exige auth (porteiro pode escanear sem login porque está autenticado pela própria URL única)

### AC7 — Tests
- [ ] `tests/unit/qr-generator.test.ts`:
  - Gera buffer PNG válido (verifica magic bytes)
  - Dimensões 1200×628 (decodificar com sharp)
  - QR é decodificável (usa `jsqr` ou similar pra validar conteúdo)
- [ ] `tests/unit/qr-token.test.ts`:
  - Token tem comprimento esperado
  - Token é URL-safe (sem `/`, `+`, `=`)
- [ ] `tests/integration/ensure-qr-for-pacote.test.ts`:
  - Primeira chamada gera, segunda chamada retorna mesma URL (idempotência)
  - Pacote inexistente lança erro

### AC8 — Smoke manual
- [ ] Criar `scripts/smoke/qr-generate.ts` que gera 1 QR, salva em `/tmp/qr-smoke.png`, abre com `open` (macOS) — visualizar no Preview pra confirmar layout

## Riscos & mitigações

| Risco | Mitigação |
|-------|-----------|
| Sharp não disponível em Alpine sem libvips | Já no projeto via Next.js; senão Dockerfile precisa adicionar `apk add vips-dev` |
| Texto SVG não renderiza com fonte custom | Usar fonte system (`-apple-system`, `Arial`) — funciona em todos os SOs |
| QR fica ilegível após compressão WhatsApp | Error correction H + tamanho 600×600 = tolerância a perdas |
