# Story 13.5 — Security headers (CSP, X-Frame-Options) + robots.txt

**Status:** Draft

**Tipo:** Security hardening
**Severidade:** MÉDIA
**Estimativa:** 0.5 dia

---

## User Story

**Como** sistema,
**eu quero** ter headers de segurança padrão e bloquear crawlers em áreas internas,
**para que** o app tenha proteção contra clickjacking, XSS e enumeração por bots.

---

## Contexto

O app não configura CSP, X-Frame-Options, X-Content-Type-Options ou Referrer-Policy além do que Clerk injeta. Também não tem `robots.txt` para bloquear crawlers em `/api/` e `/admin/`.

---

## Acceptance Criteria

### AC1 — Security headers no middleware
- [ ] `X-Frame-Options: DENY` — previne clickjacking
- [ ] `X-Content-Type-Options: nosniff` — previne MIME sniffing
- [ ] `Referrer-Policy: strict-origin-when-cross-origin` — limita referrer leaks
- [ ] `Permissions-Policy: camera=(self), microphone=()` — restringe APIs sensíveis

### AC2 — CSP refinada
- [ ] Content-Security-Policy configurada permitindo:
  - `self` para scripts/styles
  - Clerk domains (já whitelistados anteriormente)
  - `img-src self data: blob:` (fotos capturadas)
  - `connect-src self` + Meta WhatsApp (webhook não precisa, é server-side)
- [ ] Sem `unsafe-inline` para scripts (next.js usa nonce)

### AC3 — robots.txt
- [ ] `public/robots.txt` com:
  ```
  User-agent: *
  Disallow: /api/
  Disallow: /admin/
  Disallow: /super-admin/
  Disallow: /portaria/
  ```

### AC4 — Testes
- [ ] Test: response headers incluem X-Frame-Options
- [ ] Test: robots.txt acessível e bloqueia /api/
