## User Story

**Como** super-admin,
**eu quero** entrar no contexto de um condomínio específico ("impersonar") para suportar dúvidas e debugar problemas reportados pelo cliente,
**para que** eu veja exatamente o que o admin do cond vê (FR-071).

---

## Contexto

Pattern clássico SaaS: super-admin escolhe um cond e o sistema passa a operar como se ele fosse admin desse cond. Banner persistente lembra "você está impersonando — clique pra sair".

**Não é troca de role no DB** — é uma flag em cookie/session que o middleware tenant lê. Ao impersonar:
- `getTenantContext()` retorna `{ kind: 'tenant', condominioId: <impersonated>, role: 'admin' }` (em vez do super_admin real)
- Toda query passa por RLS como admin do cond
- Audit log registra (story 8.3)
