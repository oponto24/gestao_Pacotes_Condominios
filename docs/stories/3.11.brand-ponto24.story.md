## User Story

**Como** product owner Ponto 24,
**eu quero** que o app tenha a identidade visual da marca (amarelo #FDC800 + violet #7C3AED, fundo claro, mais amigável que o site de marketing),
**para que** quando eu mostrar pra cliente/condomínio no celular, o app pareça "do Ponto 24" — não um template genérico.

---

## Contexto

Pasta `40-clientes/ponto-24/05-sistemas/ponto24-marketing/` tem a identidade do site institucional:
- **Brand:** `#FDC800` (amarelo dourado)
- **Accent:** `#7C3AED` (violeta)
- **Font:** Montserrat
- **Fundo do site:** `#0A0A0A` (preto)

**Decisão crítica do user:** o **app PWA mantém fundo CLARO** (não preto). Marketing site tem visual "agressivo", o app tem visual "amigável" — branco/cinza claro de fundo, amarelo como destaque (CTAs, seleções), violet pra elementos secundários (matched/IA badges).

**Acessibilidade:**
- Amarelo `#FDC800` + texto branco = **falha WCAG** (contraste ~1.6:1)
- Amarelo `#FDC800` + texto preto = **AAA** (contraste ~13:1) ✅
- Por isso `--primary-foreground` deve ser preto/cinza escuro, não branco

**O que muda visualmente:**
- Botões primários: amarelo com texto preto
- Header das páginas: nome "Ponto 24" estilizado (logo SVG fica como tech debt — user enviar depois)
- Tela de login: logo central grande + tagline
- Manifest PWA: `theme_color = #FDC800`
- Acentos visuais (rings, badges importantes): `#FDC800` ou `#7C3AED`
- Status badges atuais permanecem (tem cores semânticas próprias por status)

## Dev Notes

### Pattern reusado
- Todos os componentes shadcn já usam tokens via CSS vars — mudança propaga automaticamente
- `next/font` é o padrão Next.js 14+ (já usado pela 3.1)

### Nova superfície
- **Primeira identidade visual da marca aplicada** — pattern pra próximas evoluções
- `<Logo>` componente isolado que pode ser substituído por SVG quando arrival

### Decisões importantes

1. **Sem fundo escuro** (decisão user explícita) — marketing site é dark, app é light pra "amigável".
2. **Amarelo como primary mas com texto preto** — contraste WCAG AAA. Amarelo + branco falha.
3. **Accent violet só em elementos não-críticos** (badges, indicadores secundários). CTAs sempre amarelo.
4. **Logo wordmark inline** até user enviar SVG (tech debt). "Ponto" cinza escuro + "24" amarelo destaca a marca sem precisar de asset.

### Riscos

1. **Botões shadcn podem não respeitar `primary-foreground` automaticamente** — verificar `Button` component, ajustar variant `primary` se necessário.
2. **Cor warning também é amarelada** (`#F59E0B`) — pode confundir com primary em algumas telas. Manter porque já tem contexto semântico (warning = alerta).
3. **Status badges existentes** usam paleta semântica (azul aguardando, verde retirado) — manter, são contextuais.
