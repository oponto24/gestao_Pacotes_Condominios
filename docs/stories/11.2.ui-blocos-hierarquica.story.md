# Story 11.2 — UI `/admin/blocos` hierarquica

**Epic:** E11 — UX Admin Refinado
**Status:** Done
**Owner:** @dev (Dex)
**Estimativa:** 0.5 dia
**Branch:** `main`
**Depende de:** 11.1

---

## User Story

**Como** admin do condominio,
**eu quero** ver meus blocos/torres organizados visualmente com contagem de unidades,
**para que** eu consiga navegar rapidamente ate as unidades de cada bloco sem precisar filtrar manualmente.

---

## Acceptance Criteria

### AC1 — API GET `/api/admin/blocos`
- [x] Lista blocos do condominio atual (tenant context)
- [x] Inclui `_count.unidades` por bloco
- [x] Suporta `?includeInativos=true`
- [x] Ordenado por `ordem ASC, nome ASC`
- [x] Guard: `requireAdmin()`

### AC2 — API POST `/api/admin/blocos`
- [x] Cria bloco com `{ nome, descricao?, ordem? }`
- [x] `condominio_id` do tenant context
- [x] Valida nome unico por condominio
- [x] Retorna 201

### AC3 — API `/api/admin/blocos/[id]`
- [x] GET: bloco + unidades com count moradores
- [x] PATCH: atualiza nome/descricao/ordem/ativo
- [x] DELETE: soft delete (ativo=false)

### AC4 — Page `/admin/blocos`
- [x] Server component com fetch via Prisma
- [x] Grid responsivo de cards (1/2/3 colunas)
- [x] Card: nome, descricao truncada, count unidades, badge ativo/inativo
- [x] Click → navega pra `/admin/blocos/[id]`
- [x] Botao "Novo Bloco" abre Sheet com form

### AC5 — Page `/admin/blocos/[id]`
- [x] Header com nome do bloco + botao editar
- [x] Tabela de unidades do bloco (identificador, moradores, status)

### AC6 — Quality gates
- [x] Lint, typecheck, test, build OK
