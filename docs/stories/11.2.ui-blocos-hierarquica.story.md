# Story 11.2 — UI `/admin/blocos` hierarquica

**Epic:** E11 — UX Admin Refinado
**Status:** Done
**Owner:** @dev (Dex)
**Estimativa:** 0.5 dia
**Branch:** `main`
**Depende de:** 11.1

---

## User Story

**Como** admin do condominio,
**eu quero** ver meus blocos/torres organizados visualmente com contagem de unidades,
**para que** eu consiga navegar rapidamente ate as unidades de cada bloco sem precisar filtrar manualmente.

---

## Acceptance Criteria

### AC1 — API GET `/api/admin/blocos`
- [ ] Lista blocos do condominio atual (tenant context)
- [ ] Inclui `_count.unidades` por bloco
- [ ] Suporta `?includeInativos=true`
- [ ] Ordenado por `ordem ASC, nome ASC`
- [ ] Guard: `requireAdmin()`

### AC2 — API POST `/api/admin/blocos`
- [ ] Cria bloco com `{ nome, descricao?, ordem? }`
- [ ] `condominio_id` do tenant context
- [ ] Valida nome unico por condominio
- [ ] Retorna 201

### AC3 — API `/api/admin/blocos/[id]`
- [ ] GET: bloco + unidades com count moradores
- [ ] PATCH: atualiza nome/descricao/ordem/ativo
- [ ] DELETE: soft delete (ativo=false)

### AC4 — Page `/admin/blocos`
- [ ] Server component com fetch via Prisma
- [ ] Grid responsivo de cards (1/2/3 colunas)
- [ ] Card: nome, descricao truncada, count unidades, badge ativo/inativo
- [ ] Click → navega pra `/admin/blocos/[id]`
- [ ] Botao "Novo Bloco" abre Sheet com form

### AC5 — Page `/admin/blocos/[id]`
- [ ] Header com nome do bloco + botao editar
- [ ] Tabela de unidades do bloco (identificador, moradores, status)

### AC6 — Quality gates
- [ ] Lint, typecheck, test, build OK
