## User Story

**Como** super-admin e admin_master,
**eu quero** que todas as mutacoes em entidades criticas sejam auditadas automaticamente com antes/depois,
**para que** eu possa rastrear exatamente o que mudou, quando, e por quem.

## Acceptance Criteria

- [x] AC1: Helper `auditedUpdate()` captura before/after automaticamente em qualquer update
  - Faz findUnique antes do update, executa update, grava diff no metadata do AuditLog
  - `metadata.before` e `metadata.after` com valores dos campos alterados
- [x] AC2: Helper `auditedDelete()` captura estado antes do soft delete
  - Grava snapshot completo em `metadata.before`
- [x] AC3: Rotas admin CRUD (moradores, unidades, setores, blocos) integram auditedUpdate/Delete
  - Sem precisar mudar schema Prisma (diffs vao no campo metadata JSON existente)
- [x] AC4: Rotas condominio (PATCH, DELETE) tambem integradas
- [x] AC5: Helper `auditedCreate()` registra criacao com snapshot do criado em metadata.after
- [x] AC6: Testes unitarios validam captura de diff correta

## Decisoes tecnicas

1. **Sem migration:** Diffs armazenados em `metadata` JSON existente (campo ja aceita qualquer estrutura)
2. **Formato:** `{ before: { campo: valorAntes }, after: { campo: valorDepois }, changed: ['campo1','campo2'] }`
3. **Granularidade:** Captura apenas campos que efetivamente mudaram (diff semantico)
4. **Entidades auditadas:** Condominio, User, Morador, Unidade, Setor, Bloco
5. **Nao auditadas:** PacoteFoto (append-only), PacoteEvento (ja e event log), WhatsAppMessage
6. **Performance:** Uma query extra (findUnique) por update — aceitavel dado volume baixo de mutacoes admin
