## User Story

**Como** porteiro/admin,
**eu quero** ver uma lista dos pacotes que ficaram em "pendente de identificação" (FR-021),
**para que** eu possa abrir cada um, casar manualmente com unidade/morador na tela 3.8 e prosseguir o fluxo.

---

## Contexto

Quando a IA (3.5) + matching (3.7) não conseguem casar pacote com unidade/morador automaticamente, ele fica em status `pendente_identificacao`. O porteiro precisa ver esses pacotes em algum lugar — caso contrário ficam invisíveis.

A 3.10 é a **caixa de saída de pendências** da portaria. Lista simples ordenada por mais antigo primeiro (urgência). Cada item linka pra `/chegada/confirmar/{id}` (3.8) onde já tem todo o form pronto.

**Já está no BottomNavBar** (story 3.1 criou o item "Pendentes" — verificar; senão adicionar nesta story).

**Fora do escopo:**
- Filtros/busca (tech debt — quando lista crescer)
- Bulk resolve (Epic 6.4 admin)
- Cancelar pacote (Epic 6.x)

## Dev Notes

### Pattern reusado
- Server page + lib helper + simples lista (igual telas /admin/*)
- Empty state pattern (já existe em outras telas admin)

### Riscos

1. **Lista grande:** se condomínio acumula 50+ pendências (admin negligente), página lenta. Limit 100 + tech debt: paginar ou filtros.
2. **Pacote pendente sem foto:** não deveria acontecer (3.4 garante foto), mas defesa: se foto null, mostrar placeholder.
