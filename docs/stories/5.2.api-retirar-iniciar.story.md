## User Story

**Como** sistema,
**eu quero** validar o `qr_token` apresentado pelo morador e retornar os dados do pacote,
**para que** a tela de confirmação (5.3) mostre foto + destinatário + setor pra porteiro confirmar a entrega (FR-041).

---

## Contexto

Endpoint chamado pela tela `/retirada/confirmar/[token]` (5.3) durante carregamento. Recebe o `qr_token` da URL, busca pacote por token (índice no schema), valida estado, retorna dados pra UI.

**Endpoint NÃO altera estado** (só valida + retorna). Estado muda no `confirmar` (5.4).

---

## Acceptance Criteria

### AC1 — Endpoint
- [x] Criar `src/app/api/pacotes/retirar/iniciar/route.ts` (token vai no body, não na URL — segurança)
- [x] `requirePorteiro()` guard
- [x] Body Zod: `{ qr_token: string().min(16).max(64) }`
- [x] Lookup `pacote.findFirst({ where: { qr_token } })` via `withTenant` (RLS garante tenant)
- [x] Validações:
  - 404 se token não existe
  - 409 `already_retirado` se status `retirado` ou `qr_consumido_em` setado
  - 409 `cancelado` se status `cancelado`
  - 409 `nao_pronto` se status diferente de `aguardando_retirada`
- [x] Retorna 200 com:
  ```json
  {
    "ok": true,
    "pacote": {
      "id", "qr_token", "status",
      "nome_destinatario_etiqueta", "complemento_etiqueta", "remetente",
      "unidade": { "id", "identificador", "bloco" },
      "destinatario": { "id", "nome" } | null,
      "setor": { "id", "nome" } | null,
      "posicao",
      "foto_storage_path"
    }
  }
  ```

  - Inclui `unidade`, `destinatario`, `setor`, foto principal
