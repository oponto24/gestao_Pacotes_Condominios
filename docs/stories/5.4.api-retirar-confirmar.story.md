## User Story

**Como** sistema,
**eu quero** registrar a retirada do pacote, marcar QR como consumido e mudar status pra `retirado`,
**para que** o pacote saia da lista de pendentes e tenha audit trail completo (FR-045/046/047).

---

## Acceptance Criteria

### AC1 — Endpoint
- [x] Criar `src/app/api/pacotes/[id]/retirar/confirmar/route.ts`
- [x] `requirePorteiro()` guard
- [x] Body Zod:
  ```ts
  {
    proprio_destinatario: boolean,
    retirado_por_terceiro?: string().min(3).max(200).nullable(),
  }
  ```
- [x] Refinement: se `proprio_destinatario=false`, `retirado_por_terceiro` é obrigatório (não null/empty)

### AC2 — Validações
- [x] 404 pacote não existe
- [x] 409 já retirado (qr_consumido_em not null OU status `retirado`)
- [x] 400 status diferente de `aguardando_retirada` (não pode retirar pacote em rascunho)

### AC3 — Atualização (transação)
- [x] Update pacote:
  - `status = 'retirado'`
  - `qr_consumido_em = now()`
  - `retirado_em = now()` (FR-045)
  - `funcionario_entregador_id = ctx.userId`
  - Se `proprio_destinatario=true`: `retirado_por_morador_id = pacote.destinatario_id`, `retirado_por_terceiro = null`
  - Se `proprio_destinatario=false`: `retirado_por_morador_id = null`, `retirado_por_terceiro = nome_digitado`
- [x] Cria evento `pacote_evento` tipo `retirado` com metadata:
  - `funcionario_entregador_id`, `proprio_destinatario`, `retirado_por_morador_id` ou `retirado_por_terceiro`

### AC4 — Idempotência
- [x] Se já existe evento `retirado` pra esse pacote: retorna 200 com `already_retirado: true`

### AC5 — Tests
- [x] `tests/unit/retirar-confirmar-validator.test.ts` (8 casos: válido, terceiro sem nome, etc)
- [x] `tests/integration/pacote-retirar-confirmar.test.ts`:
  - Retira proprio → status retirado, retirado_por_morador_id setado
  - Retira terceiro → retirado_por_terceiro setado, morador_id null
  - Falha 409 se já retirado
  - Falha 400 se status rascunho
  - Tenant isolation
  - Idempotência (2x mesma confirmação)

### AC6 — Quality gates
- [x] Lint, typecheck, test, build OK
