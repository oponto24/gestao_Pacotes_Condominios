## User Story

**Como** porteiro,
**eu quero** escanear um QR code com a câmera do celular e ver todos os pacotes pendentes daquele morador/unidade,
**para que** eu possa entregar múltiplos pacotes de uma vez sem escanear cada um individualmente.

---

## Contexto

A story 5.1 usava `<BarcodeScannerInput>` (input de texto com scanner de barcode). Na prática, o porteiro precisa da câmera traseira para ler QR codes dos moradores. Além disso, quando um morador tem 3 pacotes, não faz sentido exigir 3 scans — basta 1 scan para mostrar todos.

---

## Acceptance Criteria

### AC1 — Camera QR scanner com html5-qrcode
- [x] `RetiradaScannerClient.tsx` reescrito com `html5-qrcode`
- [x] Auto-inicia câmera traseira no mount
- [x] Vibra no scan bem-sucedido (`navigator.vibrate`)
- [x] Navega para `/retirada/confirmar/{token}` automaticamente
- [x] Fallback: seção colapsável "Digitar código manualmente"

### AC2 — Multi-pacote na confirmação
- [x] `pacote-retirada.ts` `loadPacoteByQrToken` retorna `outrosPacotes[]`
- [x] Busca todos pacotes com status `aguardando_retirada` ou `em_administracao` do mesmo morador/unidade
- [x] `RetiradaConfirmarClient.tsx` mostra cards com checkbox para cada pacote
- [x] Pacote escaneado pré-selecionado e não-desmarcável
- [x] Todos pré-selecionados por default
- [x] Badge mostra quantidade selecionada/total

### AC3 — Confirmação em lote
- [x] Botão "Confirmar entrega" chama API para cada pacote selecionado via `Promise.all`
- [x] Sucesso redireciona para `/retirada/sucesso?qtd=N`

### AC4 — Página sucesso com quantidade
- [x] `sucesso/page.tsx` aceita `?qtd=N` para mensagem plural
- [x] "3 pacotes entregues!" vs "Entregue com sucesso!"
