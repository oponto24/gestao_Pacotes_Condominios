## User Story

**Como** porteiro,
**eu quero** abrir a tela `/retirada` e escanear o QR Code que o morador apresenta no celular,
**para que** eu identifique imediatamente qual pacote ele veio buscar e siga o fluxo de confirmação (story 5.3).

---

## Contexto

A 3.x cria pacotes com `qr_token` único. A 4.x (Notificação WhatsApp — futuro) envia esse QR pro morador. Esta story (5.1) é a **porta de entrada da retirada** — onde o porteiro escaneia o QR.

**FR-040:** Funcionário deve poder iniciar a retirada escaneando o QR Code.

**Esta story entrega:**
- Tela `/retirada` minimalista: scanner ativo + input fallback de código manual
- Após scan/digitação, redireciona para `/retirada/confirmar/{qr_token}` (story 5.3 fará a tela)
- Validação: quando código não é UUID/token válido, mensagem amigável

**Fora do escopo:**
- API que valida QR e retorna pacote (story 5.2)
- Sheet de confirmação destinatário (story 5.3)
- API de confirmação final (story 5.4)
- Geração + envio do QR (Epic 4)
