# Story 13.3 — Remover PII de URLs (destinatário, QR token em query params)

**Epic:** E13 — Hardening UI/UX + Segurança Frontend (P0)
**Status:** Draft
**Owner:** @dev (Dex)
**Tipo:** Security/UX fix
**Severidade:** MÉDIA
**Estimativa:** 0.5 dia

---

## User Story

**Como** morador,
**eu quero** que meu nome não apareça em URLs do browser,
**para que** minha informação pessoal não fique em histórico do navegador, logs de servidor ou headers Referrer.

---

## Contexto

1. `/retirada/sucesso?destinatario=João Silva&qtd=3` — nome pessoal em URL
2. `/retirada/confirmar/{qr_token}` — token de retirada no path (por design, aceitar risco ou mitigar)

O item 1 é fix direto. O item 2 é aceitável (token é opaco, single-use por design), mas documentar decisão.

---

## Acceptance Criteria

### AC1 — Remover destinatário da URL de sucesso
- [ ] `/retirada/sucesso` não recebe mais `?destinatario=` como query param
- [ ] Dados de confirmação passados via estado de navegação (sessionStorage ou router state)
- [ ] Se state não disponível (refresh), mostra mensagem genérica "Entrega confirmada"

### AC2 — QR token: documentar risco aceito
- [ ] QR token no path `/retirada/confirmar/{token}` é aceito pois:
  - Token é opaco (UUID, não contém PII)
  - Token é consumido após uso (qr_consumido_em)
  - Adicionar header `Referrer-Policy: no-referrer` nas páginas de retirada

### AC3 — Testes
- [ ] Test: página sucesso funciona sem query params
- [ ] Test: Referrer-Policy header presente em páginas de retirada
