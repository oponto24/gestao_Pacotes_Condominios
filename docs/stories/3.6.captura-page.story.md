## User Story

**Como** porteiro,
**eu quero** uma tela única de chegada onde tiro a foto, biparia o código (opcional) e envio com **um toque** para o sistema,
**para que** o registro do pacote (foto + IA + rascunho) aconteça em <30s sem fricção (NFR-040, FR-010).

---

## Contexto

A 3.2/3.3/3.4 entregaram componentes isolados + um **playground** em `/chegada`. Esta story **promove o playground a tela final**: remove rótulos "playground", refina UX, cobre estados de erro e redireciona pro `/chegada/confirmar/{pacote_id}` (story 3.8).

A confirmação de dados extraídos pela IA é a **3.8**. Esta story **NÃO** mostra dados da IA — só dispara a captura e redireciona.

**Fluxo:**
1. Porteiro abre `/chegada` (já protegido por `requirePorteiro`, layout PortariaLayout da 3.1)
2. Estado inicial: PhotoCapture pronto, código vazio
3. Porteiro tira foto → Blob no estado
4. (Opcional) bipa código de barras → string no estado
5. Toca "Registrar pacote" → POST `/api/pacotes` com FormData
6. Loading 2-4s (upload + transação DB)
7. Sucesso 201 → toast verde + redirect `/chegada/confirmar/{pacote_id}`
8. Erro → toast vermelho com mensagem da API + permite retry sem perder foto/código

**Fora do escopo:**
- Tela de confirmação dos dados IA (story 3.8)
- Matching CEP+nome (story 3.7)
- Lista de pacotes pendentes (story 3.10)
- Tamanho/setor (story 3.9)

## Dev Notes

### Pattern reusado
- **Reducer pattern** da 3.3 (BarcodeScannerInput) — discriminação por estado evita bugs de useState concorrentes
- **PhotoCapture / BarcodeScannerInput** consumidos como caixas-pretas — não tocar nesses arquivos
- **POST /api/pacotes** já está pronto (3.4) — esta story só consome via `fetch`

### Nova superfície
- Primeira tela "real" da PWA (não playground)
- Primeira navegação `router.push` → estabelece pattern para 3.7-3.10

### Riscos

1. **Rota `/chegada/confirmar/{id}` ainda não existe** (é a 3.8). Redirect cai em 404. **Aceitável** nesta story — o smoke valida que o pacote foi criado, próxima story preenche destino. Documentar no smoke.
2. **Retry após erro** pode duplicar pacote se a request anterior já chegou no servidor (timeout client mas DB processou). MVP aceita — Idempotency-Key header é tech debt registrado da 3.4.
3. **Bug `removeChild` do BarcodeScanner** (resolvido na 3.3). Confirmar que não regrediu durante navigation.

### Sugestões para Dev Notes (do PO antecipadamente)

1. **Loader visual durante upload:** uploads de foto 5MB em 4G podem levar 5-10s. Adicionar progress text "Enviando foto…" → "Processando…" para feedback.
2. **Vibração táctil em sucesso:** `navigator.vibrate(200)` em mobile ao concluir registro. Touch nice. Não-bloqueante.
3. **Reset automático após redirect:** se usuário voltar (back button) pra `/chegada`, estado deve ser idle limpo. Reducer já garante porque component remounta.
