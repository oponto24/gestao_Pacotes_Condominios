# Story 13.4 — Sanitizar PII em metadata de audit log

**Epic:** E13 — Hardening UI/UX + Segurança Frontend (P0)
**Status:** Done
**Owner:** @dev (Dex)
**Tipo:** Security/Privacy fix
**Severidade:** MÉDIA
**Estimativa:** 0.5 dia

---

## User Story

**Como** super-admin,
**eu quero** que o audit log não exiba telefones, emails ou dados pessoais em diffs,
**para que** informações sensíveis dos moradores não fiquem expostas no painel de auditoria.

---

## Contexto

`AuditDiffViewer` renderiza `JSON.stringify(metadata)` completo. Quando um morador é editado, o diff before/after mostra telefone, email e outros campos PII. Além disso, `clerk_id` com pattern `pending_clerk_link_` revela arquitetura interna.

---

## Acceptance Criteria

### AC1 — Redação de campos sensíveis no viewer
- [x] `AuditDiffViewer` redacta campos: `telefone`, `email`, `clerk_id`, `contato_telefone`, `contato_email`
- [x] Campos redactados mostram `[redacted]` em vez do valor
- [x] Campos `id`, `created_at`, `updated_at` continuam filtrados (já implementado)

### AC2 — clerk_id opaco no UI
- [x] `UsersListClient` não exibe `clerk_id` diretamente
- [x] Se user está pending, mostra badge "Pendente" em vez de `pending_clerk_link_...`
- [x] Coluna `clerk_id` removida ou renomeada para "Status Clerk" com ícone

### AC3 — Testes
- [x] Test: AuditDiffViewer redacta campos sensíveis
- [x] Test: UsersListClient não renderiza clerk_id raw
